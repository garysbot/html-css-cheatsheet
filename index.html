Document Standards

There's a few helpful document standards that are required for every HTML page.
Below will remind you of what they are so you don't forget to put them in your code.

Code Explanation
<!DOCTYPE HTML> Finally lol. This tells the web browser it's HTML.
<html></html> Necessary to contain all the code.
<head></head> CSS, scripts, title, metadata goes in here.
<body></body> Main content portion of your HTML.
<div id="id-example">
<div class="class-example"> 		Attributes that make organizing and styling easier.
IDs are one-off. Classes are reusable.

CSS Setup and Syntax

Pointing to a CSS file requires the below code added to the <head> of an HTML file to link out to a CSS file

                        
                            <link href="style.css" rel="stylesheet">

CSS Fundamentals

CSS, cascading style sheets, is a programming language used to style the HTML content on a webpage.

                        
                            p {
                                border-color: #ffffff;
                                border-style: solid;
                                font-family: monospace;
                            }
                        >
  • Selector: The beginning of the ruleset used to target the element that will be styled.
  • Declaration Block: The code in-between (and including) the curly braces (`{ }`) that contains the CSS declaration(s).
  • Declaration: The group name for a property and value pair that applies a style to the selected element.
  • Property: The first part of the declaration that signifies what visual characteristic of the element is to be modified.
  • Value: The second part of the declaration that signifies the value of the property.
